# 📌 Flask Batman Project 🦇

Это демонстрационное веб-приложение на Flask, созданное для демонстрации основ современной веб-разработки: маршрутизации, валидации форм, логирования, тестирования и работы с шаблонами Jinja2.

## ✏️ 🚀 Функционал
- Главная страница: Приветственная страница с адаптивным дизайном и симметричным оформлением.
- О проекте: Страница с описанием целей и технологий.
- Форма контактов: 
- Валидация полей (проверка на пустоту и формат email).
- Защита от дублирования сообщений (паттерн PRG — Post-Redirect-Get).
- Всплывающие уведомления (Flash messages).
- Логирование: Полное отслеживание действий пользователя и ошибок сервера в реальном времени.
- Обработка ошибок: Кастомные страницы для ошибок 404 (не найдено) и 500 (ошибка сервера).

## ✏️ 🛠 Технологии
- Backend: Python 3.x, Flask
- Frontend: HTML5, CSS3, Jinja2
- Testing: Pytest
- Logging: Python Logging Module

## 📝 Система логирования

Приложение реализует комплексную систему логирования, которая:

- Записывает все события приложения и ошибки.
- Использует различные уровни логирования (INFO, WARNING, ERROR, DEBUG, CRITICAL).
- Обеспечивает цветной вывод в консоль для лучшей читаемости.
- Хранит логи в отдельных файлах(app.log для общей информации, error.log для ошибок) в общей директории `logs/`.

### Файлы логов

- **app.log**: Содержит общую информацию о работе приложения, включая действия пользователя, успешное выполнение запросов к API и информационные сообщения.
- **error.log**: Записывает все сообщения уровня `ERROR` и выше.

### Компоненты логирования

Настройка системы логирования осуществляется через следующие файлы:
- **my_logger_config.py**: Настраивает конфигурацию логирования
- **my_logging_config.yaml**: Конфигурация для различных логгеров
- **my_color_formatter.py**: Кастомный форматтер для цветного вывода в консоль

## 🧪 Тестирование

Для обеспечения стабильности приложения проект покрыт автоматическими тестами на базе фреймворка `pytest`.
Тесты проверяют:
- Доступность всех маршрутов (200 OK).
- Корректность работы валидатора email.
- Успешную отправку формы и обработку ошибок ввода.
- Работу обработчика 404 ошибки.


### Запуск тестов

Тесты можно запустить из командной строки:
- `pytest` - Краткий вариант результатов тестирования
- `pytest -v` - Более детальный вывод результатов тестирования

Также файл с тестированием можно запустить как и обычный код, через кнопку `run`.

### Примеры тестов

1. Проверяет успешную отправку формы.
```python
def test_contact_post_success(client):
    """Проверяет успешную отправку формы."""
    data = {
        "name": "Иван",
        "email": "ivan@test.com",
        "message": "Привет, это тестовое сообщение!"
    }
    # Используем follow_redirects=True, чтобы проверить сообщение после редиректа
    response = client.post('/contact', data=data, follow_redirects=True)
    assert response.status_code == 200
    assert "Спасибо, Иван! Ваше сообщение успешно отправлено." in response.data.decode('utf-8')
```
2. Проверяет отправку формы с некорректным email.
```python
def test_contact_post_invalid_email(client):
    """Проверяет отправку формы с некорректным email."""
    data = {
        "name": "Иван",
        "email": "плохой-email",
        "message": "Текст сообщения"
    }
    response = client.post('/contact', data=data)
    assert "Некорректный формат email!" in response.data.decode('utf-8')
```
